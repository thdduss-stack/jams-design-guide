## 1. 디자인 철학

### 1.1 핵심 가치

**명확함** — 비즈센터는 채용 담당자가 사용하는 도구입니다. 모든 UI는 "이것이 무엇이고, 무엇을 할 수 있는가"를 즉각적으로 전달해야 합니다.

**일관성** — 같은 의미를 가진 요소는 같은 모양을 합니다. 버튼은 버튼처럼, 링크는 링크처럼 보여야 합니다. 동일한 패턴이 반복될수록 사용자는 빠르게 학습합니다.

**절제** — 장식보다 기능을 우선합니다. 색상, 그림자, 애니메이션은 정보 전달에 도움이 될 때만 사용합니다.

### 1.2 테마

비즈센터는 **두 가지 브랜드 테마**를 지원합니다.

| 브랜드 | Primary(주색) | Accent(강조색) | 분위기 |
|--------|-------------|---------------|--------|
| **잡코리아** | 파랑 `#003CFF` | 주황 `#FF501B` | 신뢰, 전문성 |
| **알바몬** | 주황 `#FF501B` | 파랑 `#435EFF` | 활력, 친근함 |

디자인할 때는 특정 색상 값(HEX)이 아닌 **역할 기반 색상**(Primary, Accent 등)으로 지정하세요. 테마가 전환되면 색상이 자동으로 바뀝니다.

### 1.3 대상 환경

- **데스크톱 전용** (최소 1216px ~ 최대 1856px)
- 모바일/태블릿은 별도 프로젝트에서 대응
- 폰트: **Pretendard** (한글 최적화 웹폰트)

## 2. 색상 시스템

### 2.1 색상 설계 원칙

> "색상은 정보다."

색상은 장식이 아니라 **의미를 전달하는 도구**입니다. 새로운 색상을 사용하기 전에 "이 색상이 사용자에게 무엇을 말해주는가?"를 먼저 생각하세요.

### 2.2 텍스트 색상 계층

가장 중요한 것부터 덜 중요한 것 순으로 밝아집니다.

| 계층 | 색상 | Tailwind | 어떤 텍스트에 사용하나요? |
|------|------|---------|------------------------|
| **1단계 (최강조)** | Gray 900 | `text-gray-900` | 페이지 제목, 핵심 데이터 수치, 사용자가 반드시 봐야 할 정보 |
| **2단계 (강조)** | Gray 800 | `text-gray-800` | 카드 제목, 서브 제목, 중요 본문 |
| **3단계 (기본)** | Gray 700 | `text-gray-700` | 일반 본문, 라벨 |
| **4단계 (보조)** | Gray 600 | `text-gray-600` | 부가 설명, 날짜/시간 등 메타 정보 |
| **5단계 (약화)** | Gray 500 | `text-gray-500` | 플레이스홀더, 비활성 텍스트, 캡션 |
| **6단계 (비활성)** | Gray 400 | `text-gray-400` | 완전히 비활성화된 요소 |

### 2.3 배경 색상 체계

| 역할 | 색상 | Tailwind | 언제 사용하나요? |
|------|------|---------|----------------|
| **기본 배경** | White | `bg-base-white` | 페이지 전체 배경, 카드 내부 |
| **호버 배경** | BlueGray 50 | `bg-bluegray-50` | 마우스를 올렸을 때 반응하는 영역 |
| **클릭 배경** | BlueGray 100 | `bg-bluegray-100` | 누르는 순간의 피드백 |
| **구분 배경** | Gray 50 ~ 100 | `bg-gray-50` `bg-gray-100` | 영역을 시각적으로 구분할 때 |
| **스켈레톤** | Gray 200 | `bg-gray-200` | 콘텐츠 로딩 중 자리 표시 |
| **딤드 오버레이** | Base Dimed | `bg-base-dimed` | 모달 뒤 어두운 배경 |

### 2.4 의미 색상 (Semantic Colors)

특정 **상태나 결과**를 알려줄 때 사용합니다.

| 의미 | 색상 계열 | Tailwind 예시 | 사용 상황 | 예시 |
|------|---------|-------------|----------|------|
| **행동 유도** | Blue 700 | `text-blue-700` `bg-blue-700` | 클릭 가능한 요소, 링크, Primary 버튼 | "지원자 보기", "저장" |
| **성공/완료** | Green 400~600 | `text-green-600` `bg-green-100` | 성공 알림, 완료 상태 표시 | "저장되었습니다", "참여중" |
| **경고/주의** | Yellow 300~600 | `text-yellow-600` `bg-yellow-50` | 주의가 필요한 상황 | "초대 만료 임박" |
| **오류/위험** | Red 300~600 | `text-red-600` `bg-red-150` | 에러, 삭제 확인, 필수 입력 누락 | "비밀번호 불일치", "삭제" |
| **중립 정보** | Blue 100~150 | `text-blue-700` `bg-blue-100` | 안내 배경, 정보성 뱃지 | 도움말 박스, 안내 영역 |

**규칙**: 한 화면에 의미 색상이 3종류 이상 동시에 나타나면, 정보 과부하를 의심하세요.

### 2.5 테두리 색상

| 역할 | 색상 | Tailwind | 언제 사용하나요? |
|------|------|---------|----------------|
| **카드 테두리** | BlueGray 100 | `border-bluegray-100` | 카드, 목록 항목 구분 |
| **영역 구분선** | Gray 200 | `border-gray-200` | 헤더 하단, 섹션 간 구분 |
| **강조 테두리** | Blue 700 (Primary) | `border-blue-700` | 선택된 항목, 포커스 상태 |
| **에러 테두리** | Red 600 | `border-red-600` | 유효성 검사 실패 입력 필드 |

### 2.6 라벨/뱃지 색상 팔레트

상태 라벨이나 뱃지에는 **12가지 색상** 변형을 사용할 수 있습니다.

> Purple · BlueGray · Blue · Orange · Green · Mint · Gray · Yellow · Pink · Olive · Brown · Violet

각 색상은 배경(연한 톤)과 텍스트(진한 톤)가 쌍으로 제공됩니다. 새로운 상태를 추가할 때 기존에 사용되지 않은 색상을 선택하여 혼동을 방지하세요.

### 2.7 금지 사항

- 디자인 토큰에 정의되지 않은 **임의의 HEX 값 사용 지양**
- 같은 의미에 **서로 다른 색상** 사용 금지 (예: 에러를 어떤 곳은 Red, 어떤 곳은 Orange로)
- 텍스트와 배경의 **명도 대비 4.5:1 미만** 금지 (접근성)

## 3. 타이포그래피

### 3.1 설계 원칙

> "크기와 굵기만으로 계층을 만든다."

색상에 의존하지 않고, **폰트 크기**와 **굵기**의 조합만으로 정보의 중요도를 구분합니다.

### 3.2 크기 스케일

12단계 크기를 사용합니다. 각 크기에는 행간(line-height)과 자간(letter-spacing)이 자동 연동됩니다.

| Tailwind | 픽셀 | 역할 | 사용 빈도 |
|----------|------|------|----------|
| `text-36` | 36px | 히어로 타이틀 — 랜딩, 온보딩 등 감정적 임팩트가 필요한 곳 | 드묾 |
| `text-32` | 32px | 대형 타이틀 — 비어있는 대화면의 주인공 텍스트 | 드묾 |
| `text-28` | 28px | 페이지 대제목 — 긴 워딩이 필요한 최상위 제목 | 가끔 |
| **`text-24`** | **24px** | **페이지 제목** — 각 페이지의 대표 제목 | 자주 |
| **`text-20`** | **20px** | **섹션 제목** — 페이지 내 영역 구분 | 자주 |
| `text-18` | 18px | 중간 제목 — 카드 그룹, 설정 영역 제목 | 가끔 |
| **`text-16`** | **16px** | **카드 제목 / 강조 본문** — 시선을 모아야 할 본문 | 자주 |
| `text-15` | 15px | 약간 큰 본문 — 특수 상황에서 14와 16 사이 | 드묾 |
| **`text-14`** | **14px** | **기본 본문** — 모든 일반 텍스트의 기준선 | 가장 많음 |
| `text-13` | 13px | 작은 보조 텍스트 — 테이블 셀, 밀집된 정보 | 가끔 |
| **`text-12`** | **12px** | **뱃지/태그/캡션** — 작은 라벨, 부가 정보 | 자주 |
| `text-11` | 11px | 최소 텍스트 — 법적 고지, 매우 작은 주석 | 드묾 |

### 3.3 굵기 스케일

| 이름 | Tailwind | 값 | 언제 사용하나요? |
|------|---------|-----|----------------|
| **Bold (700)** | `font-bold` | 가장 굵음 | 제목, 수치 강조, 사용자 시선이 처음 가야 할 곳 |
| **Semibold (600)** | `font-semibold` | 중간 굵음 | 서브 제목, 라벨 강조, 폼 섹션명 |
| Medium (500) | `font-medium` | 약간 굵음 | 특수한 경우의 중간 강조 (거의 사용하지 않음) |
| **Regular (400)** | `font-regular` | 기본 | 본문, 설명, 플레이스홀더 — 가장 많이 사용 |

### 3.4 권장 조합 레시피

실무에서 가장 많이 쓰이는 조합입니다. 새로운 화면을 설계할 때 이 레시피부터 시작하세요.

#### 제목 계층

| 용도 | Tailwind 클래스 조합 |
|------|---------------------|
| 대제목 | `text-28 font-bold text-gray-900` |
| 페이지 제목 | `text-24 font-bold text-gray-900` |
| 섹션 제목 | `text-20 font-bold text-gray-900` |
| 카드 제목 | `text-16 font-bold text-gray-800` |
| 서브 라벨 | `text-16 font-semibold text-gray-900` |

#### 본문 계층

| 용도 | Tailwind 클래스 조합 |
|------|---------------------|
| 기본 본문 | `text-14 font-regular text-gray-900` |
| 설명 텍스트 | `text-14 font-regular text-gray-800` |
| 보조 텍스트 | `text-14 font-regular text-gray-500` |
| 강조 본문 | `text-14 font-semibold text-gray-900` |

#### 작은 요소

| 용도 | Tailwind 클래스 조합 |
|------|---------------------|
| 뱃지/태그 | `text-12 font-semibold` + (의미 색상) |
| 캡션 | `text-12 font-regular text-gray-500` |
| 포인트 수치 | `text-20 font-bold text-blue-700` |

### 3.5 금지 사항

- **Tailwind 기본 크기 클래스 사용 금지** — `text-sm`, `text-lg` 등은 이 프로젝트에서 작동하지 않습니다. 반드시 숫자 크기(`text-14`, `text-20` 등)를 사용하세요.
- 한 화면에 **5단계 이상의 크기 계층**이 나타나면 복잡도를 의심하세요.
- Bold와 색상 강조를 **동시에** 과도하게 사용하면 "모든 것이 중요해 보이는" 역효과가 생깁니다.

## 4. 아이콘 & 일러스트

### 4.1 아이콘 시스템 개요

비즈센터는 **자체 아이콘 시스템**을 사용합니다. 외부 아이콘 패키지(react-icons, heroicons 등)를 추가 설치하지 마세요.

| 종류 | 설명 | 수량 |
|------|------|------|
| **System 아이콘** | 화살표, 검색, 닫기 등 범용 UI 아이콘 | 수백 개 |
| **Special 아이콘** | 빈 상태 일러스트, AI, 매칭 등 비즈니스 특화 | 57개 |

### 4.2 아이콘 크기 가이드

| size prop | 크기 | 어디에 사용하나요? |
|-----------|------|-------------------|
| `'12'` `'14'` | 12~14px | 텍스트 옆 인라인 인디케이터 (화살표, 정렬 표시) |
| `'16'` | 16px | 작은 버튼 내부 아이콘, 입력 필드 접미사 |
| `'18'` `'20'` | 18~20px | 기본 버튼 아이콘, 리스트 항목 앞 아이콘 |
| **`'24'` (기본값)** | **24px** | 범용 아이콘, 네비게이션, 툴바 |
| `'28'` `'32'` | 28~32px | 강조 아이콘, 빈 상태의 보조 아이콘 |
| `'48'` | 48px | 빈 상태 일러스트, 대형 상태 표시 |

### 4.3 아이콘 색상 규칙

아이콘 색상은 주변 텍스트의 **의미 맥락**을 따릅니다.

| 상황 | color prop | 근거 |
|------|-----------|------|
| 일반 상태 | `'gray700'` (기본값) | 본문 텍스트와 조화 |
| 비활성/보조 | `'gray300'` `'gray500'` | 시각적으로 뒤로 물러남 |
| Primary 행동 | `'blue700'` | "클릭할 수 있음"을 암시 |
| 성공 상태 | `'green400'` ~ `'green600'` | 긍정적 결과 |
| 에러/삭제 | `'red600'` | 위험 또는 부정적 행동 |
| 역상 (다크 배경 위) | `'white'` | 가시성 확보 |

### 4.5 아이콘 버튼 크기 관계

아이콘 버튼의 크기(터치/클릭 영역)와 내부 아이콘 크기는 비례합니다.

| 버튼 영역 | 내부 아이콘 | 용도 |
|----------|-----------|------|
| 20px | 12px | 매우 작은 인라인 버튼 |
| 28px | 16px | 컴팩트 툴바 |
| 32px | 18px | 일반 아이콘 버튼 |
| 36px | 20px | 약간 큰 아이콘 버튼 |
| 40px | 24px | 주요 아이콘 버튼 |
| 48px | 24px | 대형 아이콘 버튼 (터치 최적화) |

### 4.6 금지 사항

- **새로운 아이콘 패키지 설치 금지** — 기존 시스템 아이콘으로 해결
- 아이콘만으로 의미를 전달하지 마세요 — 중요한 행동에는 **반드시 텍스트 라벨을 함께**
- 같은 아이콘을 **서로 다른 의미**로 사용하지 마세요

## 5. 공간과 여백

### 5.1 설계 원칙

> "여백은 요소 사이의 관계를 설명한다."

가까이 붙어있는 것은 관련이 있고, 멀리 떨어져 있는 것은 독립적입니다.
이 프로젝트의 간격은 **px 기반**입니다 (Tailwind 기본값과 다르므로 주의).

### 5.2 간격 스케일

| 값 (px) | Tailwind 예시 | 역할 | 구체적인 사용처 |
|---------|-------------|------|---------------|
| **2** | `gap-2` `p-2` | 미세 조정 | 거의 사용하지 않음. 1px 테두리 보정 등 |
| **4** | `gap-4` `p-4` | 긴밀한 관계 | 아이콘과 텍스트 사이, 칩 내부, 인라인 요소 |
| **6** | `gap-6` `p-6` | 작은 관계 | 작은 요소들의 내부 간격 |
| **8** | `gap-8` `p-8` | 기본 요소 간격 | 버튼 사이, 입력 필드와 도움말 텍스트, 아이콘과 라벨 |
| **10** | `gap-10` `p-10` | 약간 넓은 간격 | 기본보다 살짝 넉넉한 간격이 필요할 때 |
| **12** | `gap-12` `p-12` | 카드 내부 소그룹 | 관련 정보 묶음 내부, 태그 그룹 |
| **16** | `gap-16` `p-16` | 섹션 내 요소 | 폼 필드 간, 카드 내부 주요 영역 |
| **20** | `gap-20` `p-20` | 카드 내부 대그룹 | 카드 내 주요 섹션 사이, 카드 패딩(세로) |
| **24** | `gap-24` `p-24` | 카드 패딩 | 카드 내부 좌우/상하 여백, 리스트 항목 좌우 |
| **32** | `pt-32` `gap-32` | 섹션 시작 | 페이지 콘텐츠 상단 여백 |
| **40** | `px-40` `gap-40` | 페이지 좌우 | 페이지 양쪽 여백, 넓은 섹션 간격 |
| **48** | `gap-48` `p-48` | 큰 섹션 간격 | 관련 섹션 그룹 사이 |
| **80** | `gap-80` `pb-80` | 페이지 섹션 간격 | 완전히 독립된 섹션 사이, 페이지 하단 여백 |

### 5.3 여백 적용 공식

#### 레벨 1: 페이지

```
┌─────────────────────────────────────────────────┐
│← 40px →┌─────────────────────────────┐← 40px →│
│         │        섹션 A               │         │
│         └─────────────────────────────┘         │
│                    ↕ 80px                        │
│         ┌─────────────────────────────┐         │
│         │        섹션 B               │         │
│         └─────────────────────────────┘         │
│                    ↕ 80px                        │
│         ┌─────────────────────────────┐         │
│         │        섹션 C               │         │
│         └─────────────────────────────┘         │
│                    ↕ 80px (하단)                  │
└─────────────────────────────────────────────────┘

좌우 패딩: 40px → px-40
섹션 간격: 80px → gap-80
상단 패딩: 32px → pt-32
하단 패딩: 80px → pb-80
```

#### 레벨 2: 카드

```
┌─ 카드 ──────────────────────────────────────────┐
│  24px                                           │
│  ┌───────────────────────────────────────────┐  │
│  │ 카드 제목                    [토글 탭]     │  │
│  └───────────────────────────────────────────┘  │
│                   ↕ 20px                        │
│  ┌───────────────────────────────────────────┐  │
│  │ 콘텐츠 영역                                │  │
│  │                                           │  │
│  └───────────────────────────────────────────┘  │
│  24px                                           │
└─────────────────────────────────────────────────┘

내부 패딩: 24px → p-24
헤더-콘텐츠 간격: 20px → gap-20
콘텐츠 내 요소 간격: 12px → gap-12
```

#### 레벨 3: 리스트 항목

```
┌─────────────────────────────────────────────────┐
│ ← 24px → 항목 제목                    ← 24px → │
│ ← 24px → 항목 설명                    ← 24px → │
│          ↕ 내부 요소 간격: 8px                   │
├─ 구분선 ────────────────────────────────────────┤
│ ← 24px → 다음 항목 제목               ← 24px → │
└─────────────────────────────────────────────────┘

좌우 패딩: 24px → px-24
상하 패딩: 16px → py-16
항목 내부 요소 간격: 8px → gap-8
구분선: → border-b border-bluegray-100
```

#### 레벨 4: 헤더

```
┌─────────────────────────────────────────────────┐
│← 40px →  페이지 제목        [액션 버튼들]← 40px│
│              고정 높이: 62px                     │
├─ 구분선 ────────────────────────────────────────┤

좌우 패딩: 40px → px-40
고정 높이: 62px → h-[62px]
버튼 간 간격: 8px → gap-8
하단 구분선: → border-b border-gray-200
```

### 5.4 구분선 규격

| 종류 | Tailwind | 용도 |
|------|---------|------|
| **수평 구분선 (얇음)** | `border-b border-bluegray-100` | 카드 내부 항목 구분 |
| **수평 구분선 (강조)** | `border-b border-gray-200` | 헤더 하단, 섹션 구분 |
| **수직 구분선** | `h-12 w-px bg-bluegray-200` | 인라인 요소 구분 (예: "N건 보유 | 상품") |

### 5.5 핵심 규칙

- 간격은 **margin이 아닌 gap**으로 처리 (Flexbox/Grid 레이아웃)
- **스케일에 정의된 값만 사용** — 임의의 7px, 23px 같은 값 지양
- 간격이 좁을수록 **관련성이 높음**을 의미. 의도적으로 사용하세요

## 7. 그림자와 깊이

### 7.1 깊이 계층

그림자는 요소의 **Z축 위치(깊이)**를 표현합니다.

| 깊이 | Tailwind | 용도 |
|------|---------|------|
| **0 (바닥)** | 없음 | 일반 카드 → 테두리만으로 구분 |
| **1 (약간 뜸)** | `shadow-smallbox` | 작은 팝업, 호버 카드 |
| **2 (뜸)** | `shadow-dropdown` | 드롭다운 메뉴, 팝오버 |
| **3 (높이 뜸)** | `shadow-large` | 모달, 플로팅 패널 |

### 7.2 사용 가이드

| UI 요소 | Tailwind | 설명 |
|---------|---------|------|
| 일반 카드 | `border border-bluegray-100` | 대부분의 카드는 테두리만으로 충분 |
| 드롭다운 메뉴 | `shadow-dropdown` | 메뉴가 콘텐츠 위에 떠있음을 표현 |
| 모달 | `shadow-large` | 전체 화면 위에 떠있는 최상위 레이어 |
| 툴팁 | `shadow-tooltip-shadow-default` | 가벼운 부유 요소 |
| 버튼 (특수) | `shadow-button` | 눌렀을 때의 깊이감 (일반적으로 사용 안 함) |

### 7.3 원칙

- 그림자는 **최소한으로** 사용. 대부분의 영역 구분은 테두리로 충분합니다
- 그림자가 있는 요소는 반드시 **위에 떠있는 것** (드롭다운, 모달, 플로팅 버튼)
- 같은 깊이에 있는 요소끼리는 **동일한 그림자**를 사용

## 8. 페이지 구성

### 8.1 페이지 골격

모든 비즈센터 페이지는 다음 골격을 따릅니다.

```
┌──────────────────────────────────────────────────────┐
│  사이드 네비게이션  │        콘텐츠 영역                │
│  (고정 너비)       │                                   │
│                    │  ┌── 페이지 헤더 (62px) ────────┐ │
│                    │  │  제목          [액션 버튼들]   │ │
│                    │  ├────────────────────────────── │ │
│                    │  │                               │ │
│                    │  │     메인 콘텐츠                 │ │
│                    │  │     (스크롤 가능)               │ │
│                    │  │                               │ │
│                    │  │                               │ │
│                    │  └──────────────────────────────┘ │
└──────────────────────────────────────────────────────┘
```

### 8.2 콘텐츠 영역 규격

| 속성 | Tailwind | 이유 |
|------|---------|------|
| 최소 너비 | `min-w-[1216px]` | 가장 작은 데스크톱 해상도 대응 |
| 최대 너비 | `max-w-[1856px]` | 4K 모니터에서 과도한 확장 방지 |
| 중앙 정렬 | `mx-auto` | 큰 화면에서 좌측 치우침 방지 |
| 전체 높이 | `h-screen max-h-screen` | 사이드 네비와 함께 고정 레이아웃 |
| 수평 오버플로우 | `overflow-x-auto` | 최소 너비 이하 화면 대응 |

### 8.3 카드 구성 패턴

대시보드나 홈 화면에서 카드를 배치할 때:

```
┌─── 카드 A (67%) ──────────────┐  ┌─── 카드 B (33%) ──┐
│                               │  │                    │
│  고정 높이: 396px             │  │  고정 높이: 396px  │
│                               │  │                    │
└───────────────────────────────┘  └────────────────────┘
             ↔ 카드 간 간격: 24px
```

- 같은 행의 카드는 **동일한 높이**를 유지
- 카드 간 간격은 **24px**
- 비율은 Flex 비율로 정확하게 지정

### 8.4 빈 상태 디자인

데이터가 없을 때 빈 화면을 방치하지 마세요.

```
┌──────────────────────────────────┐
│                                  │
│          (48px 아이콘)            │
│              ↕ 16px              │
│       "데이터가 없습니다."         │
│         14px Regular Gray 500    │
│                                  │
└──────────────────────────────────┘

상하 패딩: 80px
아이콘: special_empty, 48px, Gray 300
```

### 8.5 스켈레톤 로딩 디자인

콘텐츠가 로딩 중일 때, **실제 콘텐츠와 동일한 크기의 회색 블록**으로 자리를 표시합니다.

```
┌──────────────────────────────────┐
│  ████████████  (제목 자리)        │   h-[20px] w-[120px]
│  ██████████████████  (본문 자리)  │   h-[28px] w-[300px]
│                                  │
│  모서리: rounded-4               │
│  배경: bg-gray-200               │
│  애니메이션: animate-pulse        │
└──────────────────────────────────┘
```

## 9. 컴포넌트 활용 원칙

### 9.1 기본 원칙

> "만들기 전에 찾아라."

새로운 UI 요소가 필요할 때, 다음 순서로 해결책을 찾으세요.

```
1단계: 기존 BZW 컴포넌트로 해결 가능한가?
  → YES → 사용
  → NO ↓

2단계: @jds/theme 컴포넌트로 해결 가능한가?
  → YES → 사용
  → NO ↓

3단계: 기존 컴포넌트를 조합하여 해결 가능한가?
  → YES → widgets 레이어에 조합
  → NO ↓

4단계: 신규 컴포넌트를 만든다 (디자인 팀 논의 후)
```

### 9.2 컴포넌트 카테고리별 가이드

#### 버튼

| 상황 | 어떤 컴포넌트? | 설명 |
|------|--------------|------|

| 아이콘만 있는 버튼 | BZWIconBtn | 삭제, 편집, 더보기 등 |
| 아이콘 + 텍스트 | BZWIconTextBtn | "추가하기", "편집하기" 등 |
| 토글 (켜기/끄기) | BZWToggleIconBtn | 즐겨찾기, 알림 설정 등 |

| 상황 | 어떤 컴포넌트? | 너비 가이드 |

| 일반 확인/입력 | BZWModal (480px) | 짧은 메시지, 간단한 폼 |
| 작은 확인 | BZWAlert | 예/아니오 수준의 확인 |
| 복잡한 폼 | BZWModal (640~720px) | 여러 필드가 있는 폼 |

- 720: 두 열 레이아웃이 필요한 폼
- 1040~1280: 테이블, 데이터 뷰, 복잡한 편집

| 상황 | 어떤 컴포넌트? | 설명 |

| 일반 텍스트 입력 | JDS TextField | 이름, 이메일 등 |
| 비밀번호 입력 | BZWTextField | 표시/숨김 토글 포함 |
| 드롭다운 선택 | JDS SelectBox 또는 BZWSelect | 목록에서 하나 선택 |
| 체크박스 | JDS Checkbox | 다중 선택 |
| 라디오 | JDS RadioGroup | 단일 선택 |
| 캘린더/날짜 | JDS Calendar / BZWSelectCalendar | 날짜 선택 |
| 기간 필터 | BZWPeriodFilter / BZWDatePickerRange | 시작~종료 날짜 |
| 파일 업로드 | BZWUploader | 드래그 & 드롭, 진행률 표시 |

#### 아바타 & 프로필

| 상황 | 어떤 컴포넌트? | 크기 가이드 |
|------|--------------|-----------|
| 리스트 내 프로필 | BZWAvatar (type='profile') | 24~32px |
| 상세 프로필 | BZWAvatar (type='profile') | 40~48px |
| 이니셜 표시 | BZWAvatar (type='word') | 원하는 크기 |
| 멤버 추가 버튼 | BZWAvatar (type='add') | 28~32px |
| 그룹 표시 | BZWGroupAvatar | 겹치는 아바타 행 |

### 9.3 Compound Component 패턴

복잡한 컴포넌트는 **조합 패턴**으로 설계되어 있습니다.

```
BZWModal
├── .Root        (열림/닫힘 상태 관리)
├── .Header      (제목, 설명)
├── .Body        (내용)
└── .Footer      (취소/확인 버튼)

BZWTabs
├── .Root        (값 관리)
├── .List        (탭 버튼 행)
│   └── .Trigger (개별 탭 버튼)
└── .Content     (탭별 콘텐츠)

BZWDropdown
├── .Root        (열림/닫힘 상태)
├── .Trigger     (클릭 영역)
└── .Content     (메뉴 내용)
```

이 패턴을 사용하면 **내부 구조를 유연하게 조합**할 수 있습니다. 예를 들어 Footer 없는 모달, 콘텐츠가 2개인 탭 등을 자유롭게 구성합니다.

## 10. 인터랙션 & 모션

### 10.1 설계 원칙

> "모션은 관계를 설명한다."

애니메이션은 장식이 아니라 "이것이 어디서 왔고, 어디로 가는지"를 보여주는 도구입니다.

### 10.2 인터랙션 상태

모든 클릭 가능한 요소는 **4가지 상태**를 가집니다.

| 상태 | Tailwind | 시각적 변화 |
|------|---------|-----------|
| **기본 (Default)** | `bg-base-white` | 정상 표시 |
| **호버 (Hover)** | `hover:bg-bluegray-50` | 배경색이 살짝 어두워짐 |
| **눌림 (Pressed)** | `active:bg-bluegray-100` | 더 진한 배경 피드백 |
| **비활성 (Disabled)** | `cursor-not-allowed opacity-50` | 흐릿한 색상, 커서 변경 |

전환 효과: 색상 변화 시 `transition-colors` 적용 (부드러운 전환)

### 10.3 등장 애니메이션

| 요소 | 동작 | 지속 시간 |
|------|------|----------|
| 모달 배경 | 페이드인 (투명 → 어두움) | 빠르게 |
| 모달 콘텐츠 | 스케일인 (95% → 100%) + 페이드인 | 빠르게 |
| 드롭다운 | 슬라이드 다운 + 페이드인 | 0.2초 |
| 드로어 | 우측에서 슬라이드인 | 빠르게 |
| 토스트 | 하단에서 슬라이드업 | 빠르게 |
| 페이지 콘텐츠 | 페이드인 + 약간의 스케일 | 0.8초 |

### 10.4 로딩 애니메이션

| 상황 | 애니메이션 | 설명 |
|------|----------|------|
| 콘텐츠 자리 표시 | 펄스 (깜빡임) | 스켈레톤 블록이 밝아졌다 어두워짐 |
| 긴 작업 진행 | 로딩 바 | 좌→우로 채워지는 바 (1.5초 반복) |
| 짧은 대기 | 로딩 점 | 3개의 점이 순차적으로 나타남 |
| AI 처리 중 | 그라데이션 회전 + 발광 | 파랑↔보라 그라데이션이 부드럽게 회전 |
| 데이터 로딩 | 스켈레톤 웨이브 | 좌→우로 빛이 지나가는 효과 (순차 딜레이) |

### 10.5 금지 사항

- **0.5초 이상의 UI 전환 애니메이션** 금지 — 사용자가 기다리는 느낌을 줍니다
- **깜빡임, 바운스 등 주의를 끄는 애니메이션** 남용 금지
- **모든 요소에 애니메이션 적용** 금지 — 중요한 전환에만 사용

## 11. 접근성

### 11.1 필수 규칙

| 항목 | 규칙 | 이유 |
|------|------|------|
| **이미지 대체 텍스트** | 모든 `<img>`에 alt 텍스트 필수 | 시각 장애 사용자 대응 |
| **아이콘 설명** | 의미 있는 아이콘에 접근성 텍스트 제공 | 아이콘만으로는 의미 파악 불가 |
| **색상 대비** | 텍스트-배경 명도 대비 4.5:1 이상 (WCAG AA) | 시력이 약한 사용자 대응 |
| **키보드 접근** | 모든 인터랙티브 요소를 키보드로 조작 가능 | 마우스를 사용할 수 없는 환경 |
| **포커스 표시** | 포커스된 요소가 시각적으로 구분 | 키보드 사용자의 현재 위치 파악 |
| **색상만으로 구분 금지** | 에러를 빨간색만으로 표시하지 말 것 | 색각 이상 사용자 대응 |

### 11.2 설계 팁

- 에러 상태에는 **색상 + 아이콘 + 텍스트**를 함께 사용
- 그래프/차트에서 색상 구분 시 **패턴이나 라벨**을 병행
- 폼 필드에는 반드시 **연결된 라벨** 제공
- 모달이 열리면 **포커스가 모달 안으로** 이동하고, 닫히면 원래 위치로 복귀
